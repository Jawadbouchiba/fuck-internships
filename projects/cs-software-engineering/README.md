# CS + Software Engineering Module

This combined module gives students a full path from core CS reasoning to production-ready software engineering.

## Who this is for

- Computer Science students who want practical implementation experience.
- Software Engineering students who want deeper algorithm and system-thinking proof.
- Self-taught developers who need public, structured project evidence.

## What to do first

1. Pick one `L1` project card below.
2. Open or claim an issue with the project ID (`CSSE-L1-xx`).
3. Implement one scoped deliverable.
4. Open PR and include evidence.
5. Add your reflection in `templates/contribution-log-template.md`.

## Learning outcomes

By contributing here, you should demonstrate:

- algorithmic reasoning and complexity awareness,
- clean, testable implementation,
- collaborative engineering workflow (issues, PRs, code review),
- maintainability decisions and trade-off communication.

## Milestone ladder

### L1 Beginner (single PR scope)

#### CSSE-L1-01: Algorithm explainer + runnable example

- Goal: explain one algorithm and provide minimal runnable code/demo.
- Deliverables:
  - doc section with intuition + complexity,
  - one runnable example input/output.
- Evidence:
  - linked issue + PR,
  - before/after clarity improvement.

#### CSSE-L1-02: Small bug fix + regression test

- Goal: fix a scoped bug and prove it stays fixed.
- Deliverables:
  - bug fix commit,
  - at least one regression test.
- Evidence:
  - failing/passing test proof,
  - short root-cause note.

#### CSSE-L1-03: Improve developer setup docs

- Goal: reduce setup friction for new contributors.
- Deliverables:
  - updated setup steps,
  - troubleshooting section for common errors.
- Evidence:
  - "new contributor test run" notes.

### L2 Intermediate (1-3 PRs)

#### CSSE-L2-01: Performance comparison project

- Goal: compare two implementation strategies.
- Deliverables:
  - benchmark script,
  - result table,
  - recommendation with trade-offs.
- Evidence:
  - reproducible benchmark command + outputs.

#### CSSE-L2-02: API feature with tests and docs

- Goal: ship one small API feature end-to-end.
- Deliverables:
  - implementation,
  - unit/integration tests,
  - API usage docs.
- Evidence:
  - test output,
  - sample request/response.

#### CSSE-L2-03: CI reliability improvement

- Goal: improve confidence in project changes.
- Deliverables:
  - CI check enhancement,
  - updated contributor guidance.
- Evidence:
  - workflow run link,
  - explanation of failure mode addressed.

### L3 Advanced (multi-issue initiative)

#### CSSE-L3-01: Mini-system design + implementation slice

- Goal: design and implement a small subsystem.
- Deliverables:
  - architecture proposal,
  - implementation of one critical slice,
  - test plan and validation.
- Evidence:
  - design decisions and trade-offs,
  - reviewer feedback addressed.

#### CSSE-L3-02: Refactor for maintainability

- Goal: reduce complexity while preserving behavior.
- Deliverables:
  - refactor plan,
  - incremental PRs,
  - comparison of old vs new maintainability metrics (qualitative or quantitative).
- Evidence:
  - risk assessment,
  - regression protection strategy.

- Starter API service with health checks, tests, and CI.

- Docs quality checker for onboarding consistency.

## Suggested weekly progression

- Week 1: complete one `L1` card.
- Week 2-3: complete one `L2` card.
- Week 4+: join or lead one `L3` initiative.

## Definition of done

- [ ] Issue claimed and scoped
- [ ] Deliverables complete
- [ ] PR merged after review
- [ ] Evidence attached (tests/benchmark/screenshots/logs)
- [ ] Contribution log updated

## Skill signals for recruiters

- implementation quality and test discipline
- production workflow fluency
- communication and ownership
