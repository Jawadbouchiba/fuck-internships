# Project Modules

Choose a domain and follow a concrete build path.

## Modules

- [CS + Software Engineering](cs-software-engineering/README.md)
- [Cybersecurity](cybersecurity/README.md)
- [Project Idea Bank](IDEA_BANK.md)

## What each module gives you

Every module includes:

- clear learning outcomes,
- `L1` / `L2` / `L3` milestone ladder,
- project cards with deliverables,
- evidence checklist for portfolio proof.

## Fast start (first contribution in 30-60 minutes)

1. Pick a module and one `L1` project card.
2. Open or claim an issue with project ID (example: `CSSE-L1-02`).
3. Ship a small PR that completes one deliverable.
4. Fill evidence fields in PR template.
5. Log your work with `templates/contribution-log-template.md`.

## How to choose your path

- Choose **CS + Software Engineering** if you want coding, architecture, testing, and production workflow skills.
- Choose **Cybersecurity** if you want secure coding, threat modeling, and defensive engineering skills.

## Definition of done (for any module task)

- [ ] Linked issue
- [ ] Merged PR
- [ ] Validation evidence attached (tests, screenshots, benchmark, or scan result)
- [ ] Showcase/contribution log updated
