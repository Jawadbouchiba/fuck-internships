# Cybersecurity Module

Build practical defensive-security proof through structured, reviewable contributions.

## What to do first

1. Pick one `L1` project card below.
2. Open or claim an issue with ID (`CY-L1-xx`).
3. Complete one small deliverable and submit PR.
4. Attach evidence and impact notes.
5. Log contribution in `templates/contribution-log-template.md`.

## Learning outcomes

- identify and explain security risks clearly,
- propose realistic mitigations,
- verify fixes using security checks,
- communicate security impact to non-security contributors.

## Milestone ladder

### L1 Beginner

#### CY-L1-01: Secure coding guideline page

- Deliverable: one focused guideline page (for example input validation or auth handling).
- Evidence: issue + PR + before/after risky pattern example.

#### CY-L1-02: Threat model starter template

- Deliverable: reusable threat model template plus one sample.
- Evidence: template file + sample scenario + mitigation notes.

#### CY-L1-03: Security issue triage note

- Deliverable: triage one security-related issue with severity rationale.
- Evidence: issue comment + risk matrix justification.

### L2 Intermediate

#### CY-L2-01: Dependency scan hardening guide

- Deliverable: docs/workflow recommendations for dependency vulnerability scanning.
- Evidence: workflow references + triage process.

#### CY-L2-02: Security review checklist for PRs

- Deliverable: PR review checklist focused on common weaknesses.
- Evidence: checklist usage example in one review.

#### CY-L2-03: Remediation playbook card

- Deliverable: step-by-step remediation for one vulnerability class.
- Evidence: repro + fix + verification approach.

### L3 Advanced

#### CY-L3-01: Security initiative proposal

- Deliverable: scoped proposal for improving project security posture.
- Evidence: threat/risk baseline, staged rollout, success criteria.

#### CY-L3-02: Incident response simulation kit

- Deliverable: mock incident scenario + response checklist + postmortem template.
- Evidence: simulation walkthrough and reflection.

## Build ideas

- OWASP-top-10-aligned examples for contributors.
- Secret handling checklist for open-source repos.
- Secure defaults guide for starter services.
- Security logging and alerting recommendation pack.
- Vulnerability triage board template.

## Definition of done

- [ ] Linked issue and scoped objective
- [ ] Merged PR
- [ ] Risk and mitigation clearly explained
- [ ] Verification evidence attached
- [ ] Contribution log updated
