# fuck-internships

`fuck-internships` is an open-source community for students and early-career developers to build real, reviewable experience through contributions.

If internships are hard to get, you can still prove your skills with shipped work, peer reviews, issue triage, docs improvements, and project leadership in public.

## Why this exists

Many students are capable but blocked by internship access. This repo creates an alternative path:

- make meaningful contributions,
- get feedback from maintainers,
- document impact with evidence you can show recruiters.

## How it works

1. Pick a contribution track in [`docs/contribution-tracks.md`](docs/contribution-tracks.md).
2. Pick your role path from [`projects/README.md`](projects/README.md):
   - CS + Software Engineering: [`projects/cs-software-engineering/README.md`](projects/cs-software-engineering/README.md)
   - Cybersecurity: [`projects/cybersecurity/README.md`](projects/cybersecurity/README.md)
3. Start with a beginner task from [`docs/onboarding/first-issues.md`](docs/onboarding/first-issues.md).
4. Submit a PR using the template.
5. Record your work using templates in [`templates/`](templates/).
6. Publish your contribution evidence in [`showcase/`](showcase/).

## Contribution tracks

- Documentation and learning resources
- Code contributions
- Issue triage and project operations
- Peer support and mentoring

Each track has quality criteria and portfolio mapping.
