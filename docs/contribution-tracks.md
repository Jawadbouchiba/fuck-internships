# Contribution Tracks

## 1) Documentation Track

Work examples:

- improve guides,
- add onboarding docs,
- clarify setup instructions.

Acceptance criteria:

- improves learner or contributor outcomes,
- technically accurate,
- linked to issue and PR.

Portfolio mapping:

- technical communication,
- developer experience design.

## 2) Code Track

Work examples:

- fix bugs,
- add small features,
- improve tests/tooling.

Acceptance criteria:

- scoped and testable,
- reviewed and merged,
- no regressions introduced.

Portfolio mapping:

- implementation quality,
- testing and debugging practices.

## 3) Triage & Operations Track

Work examples:

- reproduce issues,
- label/scope backlog,
- close stale or resolved threads.

Acceptance criteria:

- clear reproduction notes,
- accurate labeling,
- measurable backlog hygiene.

Portfolio mapping:

- product thinking,
- project operations.

## 4) Peer Support & Mentoring Track

Work examples:

- review PRs,
- answer newcomer questions,
- improve contributor success docs.

Acceptance criteria:

- feedback is specific and respectful,
- guidance leads to merged improvements.

Portfolio mapping:

- collaboration,
- leadership.
