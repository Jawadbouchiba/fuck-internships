# How It Works

## Goal

Turn open-source contributions into clear evidence of skills, ownership, and impact.

## Choose your domain path

Start by selecting a module in `projects/`:

- `projects/computer-science/README.md`
- `projects/software-engineering/README.md`
- `projects/cybersecurity/README.md`

Each module gives starter projects and role-specific deliverables.

## Contribution to evidence pipeline

1. **Pick scoped work** from issues or propose your own.
2. **Do the work publicly** in issue discussion, commits, and PR.
3. **Get reviewed** by peers or maintainers.
4. **Merge and document impact** using templates in `templates/`.
5. **Publish your showcase entry** in `showcase/contributors/`.

## What counts as evidence

- merged PR links,
- meaningful review feedback given or addressed,
- issue triage outcomes,
- before/after outcomes,
- lessons learned and next steps.

## Recruiter-facing value

Public contribution history demonstrates:

- consistency,
- communication,
- code quality habits,
- ability to collaborate in real workflows.
