# First Issues Guide

Start with tasks tagged `good first issue`.

## Role-specific starting points

### Computer Science students

- improve algorithm explanations,
- add complexity analysis notes,
- contribute tests/benchmarks for data structures tasks.

### Software Engineering students

- fix scoped bugs with tests,
- improve CI/repository automation,
- improve developer onboarding docs.

### Cybersecurity students

- document secure coding patterns,
- propose or improve threat-model templates,
- add security-check guidance in workflows.

## Before claiming an issue

- Read issue context and linked files.
- Confirm expected output.
- Ask clarifying questions in the issue if needed.

## Suggested beginner tasks

- improve one documentation page,
- add one missing template field,
- fix one small tooling/config inconsistency,
- triage one unlabeled issue with evidence.

## Definition of done

- PR is merged,
- template evidence completed,
- contribution recorded in showcase.
