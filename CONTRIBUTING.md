# Contributing Guide

Thanks for contributing to `fuck-internships`.

## Contribution standard

Your contribution should be:

- useful to students,
- reviewable in public,
- linked to an issue or documented objective,
- recorded with evidence using templates in `templates/`.

## Workflow

1. Find or open an issue.
2. Comment that you want to work on it.
3. Create a branch and implement the change.
4. Open a PR with clear context and evidence.
5. Address review feedback.
6. After merge, update your showcase entry.

## PR checklist

- [ ] Linked issue
- [ ] Clear summary of change
- [ ] Tests/docs updated if needed
- [ ] Evidence fields completed in PR template
- [ ] Respectful communication
